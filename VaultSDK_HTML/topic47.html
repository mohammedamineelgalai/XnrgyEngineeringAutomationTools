Staring in Vault 2014, plug-ins can be loaded from the default App Store folders.  This document will describe how to configure your bundle and PackageContents.xml file so that Vault can load your plug-in.

Vault requirements:

  • Vault plug-ins must have "Vault" as the platform name.  Names of specific products, like "Vault Workgroup", will be ignored.
  • The SeriesMin and SeriesMax values must be the same.  Vault plug-ins are tied to a specific release and cannot span multiple releases.
  • SeriesMin and SeriesMax values must be "V2014" to work with Vault 2014.
  • There should be a ComponentEntry that points to the .vcet.config file.  ComponentEntries for other file types, such as DLL, will be ignored.
  • The plug-in DLL needs to be in the same folder as the .vcet.config file.
  • See the Getting Started section for information on the .vcet.config file content.

 

See the App Store guidelines for requirements and guidelines not specific to Vault.

 

Example PackageContents.xml:

<?xml version="1.0" encoding="utf-8"?>
<ApplicationPackage SchemaVersion="1.0" AutodeskProduct="Vault" ProductType="Application" Name="JobProcessorApiSamples" AppVersion="1.0.0" Description="JobProcessorApiSamples" AppNameSpace="appstore.exchange.autodesk.com" OnlineDocumentation=" " ProductCode="{02718320-9C62-48A0-B0B2-942F491A5E92}" UpgradeCode="{63FBABDF-C774-4BD9-95EF-60A2B23D8145}">
  <CompanyDetails Name="MyCompany" Phone=" " Url=" " Email="john.doe@mycompany.com" />


  <Components Description="MyCompany.JobProcessorApiSamples.bundle parts">
    <RuntimeRequirements Platform="Vault" SeriesMin="V2014" SeriesMax="V2014" />
    <ComponentEntry AppName="JobProcessorApiSamples" Version="1.0.0" ModuleName="./JobProcessorApiSamples.vcet.config" />
  </Components>

</ApplicationPackage>
