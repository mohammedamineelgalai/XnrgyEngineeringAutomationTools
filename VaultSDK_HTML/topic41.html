Glossary Item Box

A Vault client will work with only one set of web services.  However the Vault server may support multiple web service version side by side.  Here is a chart explaining what versions are supported for a given server release.

Vault 2012 and earlier Support curent web service API only.
Vault 2013 Support for 2012 and 2013 web services.
Vault 2014 Support for 2012, 2013 and 2014 web services.
Vault 2015 Support for 2013*, 2014*, and 2015 web services

*ItemService and PackageService are disabled for Vault 2015 server compatibility layers

For plug-ins, there is no compatiblity.  For example, a custom command in Vault Explorer 2013 will need to be updated in order to work with Vault Explorer 2014. 

Service Packs and Point Releases

Clients are compatible with different services packs of the ADMS.  For example, a client written against Vault 2011 will work against Vault 2011 SP1.

Point releases do not have compatible APIs.  For example, a client written against ADMS 4.0 will not work against ADMS 4.5.

The Information Service

The Information Service is the one service that will work for all versions of ADMS from R5 and up.  The purpose of the service is to grab information about the ADMS, including the version.  GetSystemProducts allows a client to pre-check the server version first to determine if its compatible.

Updating to a New ADMS Version

Since the API is not backwards compatible, here are the recommended steps for updating a client to a new version of ADMS.

  1. Open your project.
  2. Update your web references and compile.
  3. If you get build errors, open the SDK documentation to figure out how to proceed:                                                                       
    - If the parameters have changed for a function, look up the help document for that function for an explanation of the new parameters.
    - If the function is missing, look in the "Changes" page to see what new function should be used.
  4. Once everything compiles, run and test.
