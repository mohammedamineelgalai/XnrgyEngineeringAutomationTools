Getting Started > How to customize Vault Explorer

Glossary Item Box

There are various ways to customize Vault Explorer through the API.  You can...
  • Add custom commands.
  • Add custom tabs.
  • Hide existing commands.
  • Subscribe to pre and post events when any Vault Explorer commands are executed.

Availability
This feature is not available in base Vault.  Vault Workgroup, Vault Collaboration or Vault Professional is required.


Getting Started
The first thing is to set up your project in Visual Studio or other IDE.  You want to create a library project (DLL) and it has to use the .NET framework. 

The next step is to create a public class which implements the IExplorerExtension interface.  This step requires you to fill in the logic for a bunch of functions.  Depending on what you want to do you may be able to leave some of the functions empty or just return null.  See the IExplorerExtention documentation for more details on the functions you need to implement.

There are  5 assembly attributes you need to have in your code:
  1. AssemblyCompany - The name of your company. 
  2. AssemblyProduct - The name of the product. 
  3. AssemblyDescription - A description of your product.
  4. ExtensionId - A unique GUID for your product.  Visual Studio has a tool that can generate one for you.  Do not copy a GUID from another extension.
  5. ApiVersion - A string indicating which API schema your product works with.  Enter in "19.0" to work with Vault 2026.
Items 1-3 are provided by Visual Studio in the AssemblyInfo file.  You just need to check that they have accurate data.  Items 4 and 5 need to be created by you.

Once you have everything coded and built, it's time to deploy it.  To deploy you create a folder for your extension under %ProgramData%/Autodesk/Vault 2026/Extensions/.  You can name your folder whatever you want, but it's probably best to name it the same as your product name.  For example, if you DLL is called HelloWorld, you would deploy to a folder called "%ProgramData%/Autodesk/Vault 2026/Extensions/HelloWorld/".
You need to deploy your DLL, any referenced DLLs and any other files you may need to live alongside the DLL.  You also need a .vcet.config file to go in your folder (see below).

NOTE: The %programData% variable may not be set for all operating systems.  In Windows XP and 2003, this is usually "C:\Documents and Settings\All Users\Application Data".  In Vista and above, this is usually "C:\ProgramData".

NOTE:  You should not deploy Autodesk.Connectivity.Explorer.Extensibility.dll, Autodesk.Connectivity.WebServices.dll or Autodesk.Connectivity.Extensibility.Framework.dll since they are already part of the Vault Explorer runtime.

Once everything is deployed, you can start (or restart) Vault Explorer to see your customization in action.


The .vcet.config File
This files tells the Vault extensions framework how to load your extension.  It's an XML file with a specific syntax.  Here is what the file should look like for a Vault Explorer extenstion.
<configuration>
  <connectivity.ExtensionSettings3>
    <extension
      interface="Autodesk.Connectivity.Explorer.Extensibility.IExplorerExtension, Autodesk.Connectivity.Explorer.Extensibility, Version=31.0.0.0, Culture=neutral, PublicKeyToken=215b0879a7566597"
      type="MyNamespace.MyClassName, MyAssemblyName">
    </extension>
  </connectivity.ExtensionSettings3>
</configuration>

You should replace "MyNamespace.MyClassName" with the full class name if the IExplorerExtension implementation class.  And you should replace "MyAssemblyName" with the assembly name of your DLL.
