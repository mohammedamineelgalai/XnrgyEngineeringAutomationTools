Knowledge Base > Clic Licensing with the Vault SDK

Glossary Item Box

Starting with Vault 2017, Vault licensing is moving to a new framework named Clic. Part of moving to the Clic framework requires that licensing be performed in the Vault SDK. In previous releases of Vault, all licensing was done on the server side and this was mostly transparent to end users of the Vault SDK. If the Vault server was unable to obtain a license, licensed web service calls were no longer allowed.

When moving to the new Clic framework, the goal was to keep as much of this same behavior as possible. In a large part, this was successful. There are no new APIs that are required to call to obtain a license. The SDK will attempt to obtain a license automatically and it will also enforce that a license is obtained before allowing any licensed web service calls. There are a few areas of change that will require attention when migrating an application to the Vault 2017 SDK, though.

New Components

One of the most obvious changes is in inclusion of new DLLs in the SDK. Specifically, AdskLicensingSDK_5.dll and Autodesk.Connectivity.LicenseWrapper.dll have been introduced.

Bitness

The Vault SDK support x64 platform only.

License Types and the Vault Explorer Requirement

The main change is in how you inform the SDK of which version of the Vault License your app needs to obtain. Vault comes in several different flavors: Vault Basic, Vault Workgroup, and Vault Professional. The Vault SDK will determine which license type to obtain based on which flavor of Vault is installed on the client machine. This differs from legacy behavior in that the license type is not determined by the type of server you are attempting to log in to. Because the license type is pre-defined, when attempting to log in to the wrong type of server, you will get a licensing error due to a license type mismatch.

Applications that are specific to a flavor of Vault can override this machine-wide setting by specifying the license type they need to aquire via their app.config file. Below is a minimal example of an app.config that specifies a Vault Professional license:

Sample App.config Copy Code

<?xml version="1.0"?>

<configuration>

<configSections>

<section name="Licensing" type="Autodesk.Connectivity.WebServices.LicensingSection, Autodesk.Connectivity.WebServices"/>

</configSections>>

<Licensing edition="Professional"/>

</configuration>

Server Forward Compatibility and Legacy APIs

The Vault server maintains the web service APIs for the previous two releases to allow legacy Vault applications to continue to connect to it in a feature named server forward compatibility.
